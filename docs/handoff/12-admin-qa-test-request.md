## QA 테스트 항목

### 1. 기본 접근

- `/admin` 접속이 되는지 확인합니다.
- 새로고침해도 화면이 유지되는지 확인합니다.
- 브라우저 콘솔에 새 오류가 발생하는지 확인합니다.

### 2. 사이드바 메뉴

아래 메뉴가 모두 클릭되고 해당 화면으로 전환되는지 확인합니다.

- 전체 대시보드
- 사용자별 사용량
- 토큰/비용 분석
- 토론 기록 관리
- 요금 계산 설정

### 3. 데이터 0건 상태

현재 실제 토론 기록이 없을 수 있으므로, 0건 상태에서도 아래를 확인합니다.

- 대시보드 수치가 0으로 표시되는지
- 사용자별 사용량 표 구조가 깨지지 않는지
- 토큰/비용 분석 영역이 빈 상태 안내를 표시하는지
- 토론 기록 관리가 빈 상태 안내를 표시하는지
- 화면이 비어 보이지 않고 패널 구조가 유지되는지

### 4. 본문 미노출 정책

사용자별 사용량과 토론 기록 상세에서 사용자 입력 본문이 직접 노출되지 않아야 합니다.

확인 문구:

```text
관리자 화면 정책상 표시하지 않음
```

또는 같은 의미의 안내가 표시되어야 합니다.

### 5. 요금 계산 설정

- OpenAI, Claude, Gemini 카드가 표시되는지 확인합니다.
- 모델명, 입력 단가, 출력 단가 입력칸이 표시되는지 확인합니다.
- 값을 변경한 뒤 새로고침해도 유지되는지 확인합니다.
- 기본값 복원 버튼이 동작하는지 확인합니다.

### 6. 실제 기록 생성 후 재검수

실제 토론 기록이 1건 이상 생긴 뒤 아래를 다시 확인합니다.

- 사용자별 주제 수 집계
- 입력 일자 표시
- 제목 표시
- 상태 표시
- 추정 토큰/비용 표시
- 토론 기록 상태 변경
- 토론 기록 삭제
- 기록 상세 보기
- 본문 미노출 유지

## QA에서 발견 시 분류 기준

- Critical: `/admin` 접근 불가, 빌드 실패, 화면 전체 렌더링 실패
- High: 메뉴 전환 실패, 요금 설정 저장 실패, 기록 상태 변경/삭제 실패
- Medium: 본문 미노출 문구 누락, 빈 상태 안내 부족, 모바일 일부 깨짐
- Low: 문구, 간격, 색상, 세부 스타일

## 2026-05-12 QA 전달 후 어드민 채팅방 확인

QA 채팅방 중간 기록 중 어드민 채팅방에 해당하는 항목을 확인했습니다.

### 어드민 채팅방 해당 항목

- `demo03` 전용 어드민 접근 정책
- `/admin` 최종 클릭 QA
- 전체 대시보드
- 사용자별 사용량
- 토큰/비용 분석
- 토론 기록 관리
- 요금 계산 설정
- 사용자 입력 본문 비노출 정책
- 실제 토론 기록 생성 후 어드민 기록 관리 확인

### 확인 완료

- `middleware.ts`에 기본 관리자 ID가 `demo03`으로 설정되어 있습니다.
- `ADMIN_USER_ID` 환경변수가 있으면 기본값 대신 해당 값을 사용할 수 있습니다.
- `demo03`이 아닌 로그인 사용자는 `/admin` 접근 시 홈으로 이동하도록 되어 있습니다.
- 비로그인 사용자는 `/admin` 접근 시 로그인 화면으로 이동하도록 되어 있습니다.
- `components/AdminConsole.tsx`의 기록 상세 영역에는 사용자 입력 본문 대신 `관리자 화면 정책상 표시하지 않음` 안내가 표시됩니다.

### 아직 보류

- 현재 로컬 `http://127.0.0.1:3043/admin` 서버 연결이 되지 않아 실제 브라우저 클릭 검수는 보류입니다.
- `demo03` 실제 로그인 후 `/admin` 진입 확인은 로컬 서버 안정화 후 다시 진행해야 합니다.
- 실제 토론 기록이 1건 이상 생성된 뒤 상태 변경, 삭제, 상세 보기, 사용자별 사용량 집계를 다시 확인해야 합니다.
