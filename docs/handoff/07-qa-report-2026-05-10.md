검수일: 2026-05-10

## 1. 수정 요약

이번 수정의 핵심은 최근 토론을 눌렀을 때 멈춘 화면처럼 보이는 문제였습니다.

원인:

- `running` 상태의 토론이 중간에 끊겨 `속마음` 이벤트만 일부 저장된 경우가 있었습니다.
- 기존 로직은 이벤트가 1개라도 있으면 자동 스트리밍을 다시 시작하지 않았습니다.
- 그래서 최근 토론 클릭은 되었지만 결과 화면에서 토론이 멈춘 것처럼 보였습니다.

수정:

- `running`이고 `finalReport`가 없으면 이벤트가 일부 있어도 자동으로 재시작합니다.
- 사용자가 직접 중단한 `paused` 상태는 자동 재시작하지 않습니다.

## 6. 개발팀 전달 주의사항

- `running` 상태 복구 로직은 정식 DB 버전에서도 반드시 유지해야 합니다.
- 토론 이벤트가 일부 저장되었다고 완료로 보면 안 됩니다.
- 완료 기준은 `finalReport` 존재 여부입니다.
- 사용자가 직접 중단한 `paused`는 자동 재시작하지 않습니다.
- fallback은 단순 비상 문장이 아니라 질문 유형별 맥락을 유지해야 합니다.
