# Copywriting and Content Design Notes

AI Talk Talk의 문구는 친근하지만 가볍게 흩어지면 안 된다.

## 문구 톤

- 쉽고 짧게 쓴다.
- 사용자가 지금 무엇을 하면 되는지 바로 알게 한다.
- 기능 설명보다 행동 유도를 우선한다.
- 불필요한 전문 용어를 줄인다.
- 귀여운 캐릭터 톤은 유지하되, 서비스 신뢰감은 잃지 않는다.

## 로그인 화면 문구 기준

- 브랜드: `AI Talk Talk`
- PWA 설치명: `AI Talk`
- 로그인 화면은 테스트 서비스임을 숨기지 않는다.
- “지정된 3명만 사용하는 테스트 공간입니다.”처럼 짧고 명확하게 쓴다.

## 입력창 문구

- placeholder는 실제 행동을 알려야 한다.
- 예: `아이디를 입력하세요`, `비밀번호를 입력하세요`
- placeholder만으로 필드 의미를 대신하지 않고, 라벨도 함께 둔다.

## 오류 문구

- 사용자를 탓하지 않는다.
- 무엇이 문제인지, 다음에 무엇을 하면 되는지 말한다.
- 예: `아이디 또는 비밀번호를 다시 확인해주세요.`

## 카피 검수 질문

- 한 번 읽고 바로 이해되는가?
- 버튼 문구가 행동을 정확히 말하는가?
- 사장님 시연 화면에서 어색한 표현이 없는가?
- AI Talk Talk 이름 정책을 지키는가?

## 저작권과 이미지 출처

- 방실장님이 제공한 이미지 또는 프로젝트 안에 명시적으로 저장한 이미지 사용을 기본으로 한다.
- 외부 이미지를 임의로 가져와 서비스 화면에 넣지 않는다.
- 새 이미지가 필요하면 파일 출처, 용도, 교체 경로를 기록한다.
