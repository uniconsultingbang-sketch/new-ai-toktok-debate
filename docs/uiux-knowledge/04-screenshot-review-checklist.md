# Screenshot Review Checklist

UIUX 채팅방은 이 체크리스트를 통과하기 전까지 완료라고 말하지 않는다.

## 캡처 전 확인

- 현재 URL이 의도한 화면인지 확인한다.
- 로그인 화면을 볼 때는 비로그인 상태인지 확인한다.
- CSS 파일이 200으로 로딩되는지 확인한다.
- 기준 이미지 파일과 현재 화면 캡처를 모두 준비한다.

## 필수 모바일 크기

- 360x800
- 390x844
- 412x915

## 로그인 화면 체크

- 로고가 좌우로 잘리지 않는가?
- Beta 영역이 오른쪽 끝에 붙어 답답하지 않은가?
- 서브카피가 너무 작거나 멀리 떨어져 있지 않은가?
- 로봇 이미지가 중심에 있고 좌우가 균형 있는가?
- 이미지 자체 배경과 화면 배경이 충돌하지 않는가?
- 아이디/비밀번호 라벨과 입력창 간격이 자연스러운가?
- 입력창 좌우 패딩이 비슷한가?
- 로그인 버튼이 화면 안에 안정적으로 보이는가?
- 하단 안내 카드가 너무 무겁거나 잘리지 않는가?

## 홈 화면 체크

- 상단 메뉴, 브랜드, 계정 버튼의 균형이 맞는가?
- 인사말과 주제 문구가 모바일에서 자연스럽게 줄바꿈되는가?
- 로봇 영역이 너무 작거나 너무 크지 않은가?
- 하단 입력창이 손가락으로 쓰기 편한가?

## 토론방 체크

- 화자 아이콘과 말풍선이 겹치지 않는가?
- Claude/Gemini 왼쪽, GPT 오른쪽 배치가 자연스러운가?
- 긴 답변의 줄간격과 말풍선 폭이 답답하지 않은가?
- 입력창이 하단에서 잘리지 않는가?

## 완료 보고 조건

- 적어도 390x844 캡처를 직접 확인했다.
- 작은 폭 360x800에서 가로 잘림을 확인했다.
- 기준 이미지와 다른 점을 말로 설명할 수 있다.
- 남은 차이가 있으면 방실장님께 숨기지 않고 말한다.
