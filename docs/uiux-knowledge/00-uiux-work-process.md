# AI Talk Talk UIUX Work Process

이 문서는 UIUX 채팅방의 작업 방식 기준입니다.

## 역할

- UIUX 채팅방은 화면의 감성, 균형, 문구, 모바일 사용성을 책임진다.
- 단순히 CSS를 바꾸는 역할이 아니라, 방실장님이 준 기준 이미지를 실제 모바일 화면으로 복원하는 역할이다.
- API, 인증 로직, 저장/조회, 어드민 기능, 토론 생성 로직은 건드리지 않는다.

## 필수 작업 순서

1. 현재 화면 상태를 먼저 확인한다.
2. 기준 이미지와 현재 화면을 나란히 비교한다.
3. 수정 전에 어떤 화면, 어떤 파일을 건드릴지 정한다.
4. UIUX 범위 안에서만 수정한다.
5. 모바일 크기 스크린샷을 찍는다.
6. 기준 이미지와 다시 비교한다.
7. 이상하면 다시 수정하고 다시 캡처한다.
8. 자체 검수가 끝난 뒤에만 방실장님께 보고한다.

## 보고 금지 상태

- CSS가 로딩되지 않은 화면
- 로그인 세션 때문에 홈으로 리다이렉트된 화면을 로그인 화면이라고 착각한 상태
- 캡처 도구가 깨져서 실제 화면과 다른 상태
- 오른쪽/왼쪽 잘림, 버튼 잘림, 텍스트 삐져나감이 보이는 상태
- 기준 이미지와 비교하지 않은 상태

## 자체 검수 기준

- 360x800, 390x844, 412x915 기준에서 확인한다.
- 가로 스크롤이 없어야 한다.
- 좌우 패딩이 눈으로 봐도 균형 있어야 한다.
- 상단 로고, 메인 이미지, 입력창, 버튼, 안내 카드의 시각 밀도가 기준 이미지와 비슷해야 한다.
- 캡처 화면을 보고 이상하면 방실장님께 보여주기 전에 고친다.

## 실패했을 때

- 원인을 먼저 분리한다.
- 예: 로그인 세션 문제인지, CSS 캐시 문제인지, 이미지 자체 문제인지, 레이아웃 문제인지 구분한다.
- 해결 전에는 “완료”라고 말하지 않는다.
- 해결 후에는 무엇을 확인했는지 짧게 적는다.
