## 2026-05-13 - AI 응답 반복 및 부실 결론 방지 보강

- 기존 저장 기록에서 AI 발언 첫머리에 같은 질문 문장이 반복되어 보였습니다.

- AI 응답이 실패했을 때 fallback 발언을 여러 번 이어 붙이면 실제 토론처럼 보이지만 내용은 반복되고 결론도 약해질 수 있습니다.

- 오류 안내는 기존 문구를 유지합니다.
  - `토론을 정리하는 중 AI 응답이 불안정합니다. 입력하신 안건은 저장되어 있습니다. 잠시 후 다시 시도해 주세요.`
- QA/개발 확인용 `dev=quick` 흐름은 기존처럼 fallback 전체 생성이 가능하게 유지했습니다.

### 확인 결과

- `npx.cmd tsc --noEmit` 통과.
- `npm.cmd run build`는 다른 채팅방의 Next dev 서버가 같은 `.next` 폴더를 동시에 쓰면서 산출물 누락 오류가 반복되어 안정 검증을 완료하지 못했습니다.
- 로컬 3010은 dev 서버로 다시 실행해 두었습니다.
- `dev=quick` API 기준으로 최종 결론이 생성되는 것은 확인했습니다.

### 남은 확인

- 실제 AI 키가 있는 Vercel production에서 새 토론을 생성해 반복 문장과 결론 품질을 다시 확인해야 합니다.
- 기존 저장 기록은 과거 결과이므로 새로 생성한 토론으로 확인해야 합니다.

## 2026-05-13 - QA `.next` 산출물 누락 이슈 확인

### 전달받은 이슈

- QA에서 임의 질문 6개로 하드코딩 위험을 확인했습니다.
- 5개 질문은 정상 응답했고, 과거 예시 답변이 섞이지 않았습니다.
- 6번째 질문에서 500 오류가 발생했고, 로그상 원인은 답변 하드코딩이 아니라 `.next` 산출물 누락이었습니다.
- 누락으로 전달받은 파일은 `.next/server/app-paths-manifest.json`, `.next/routes-manifest.json`, `webpack-runtime.js`입니다.

### 확인한 내용

- 확인 시점의 3010 서버는 개발 서버 형태로 떠 있었고, `.next/BUILD_ID`가 없는 상태였습니다.
- Next 개발 서버는 테스트 중 `.next` 산출물을 계속 재생성할 수 있어, 클릭 QA 중 특정 순간에 manifest 파일이 비는 오류가 날 수 있습니다.
- 로컬 QA처럼 안정성이 중요한 클릭 검수는 개발 서버보다 production build 후 `next start -p 3010` 방식이 안전합니다.

### 조치

- 3010 프로세스를 종료했습니다.

- 라운드 제목의 `수요`, `보안·운영`, `조직 역량` 같은 단어가 서로 섞이지 않도록 쟁점 분류 우선순위를 조정했습니다.
